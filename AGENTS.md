## Project Overview

Effect Language Service (TypeScript-Go) is a wrapper and small patch around TypeScript-Go to build the Effect Language Service and provide Effect-TS diagnostics. This project targets **Effect V4** (codename: "smol") primarily, and then effect v3 as well.

## Readme file
The project README.md contains a table that lists supported rules, if they are ported to go, the default severity, list and status of porting quickfixes, etc... Be sure to keep the readme up to date with changes.

## Submodules

The repo contains submodules and codegen; before any change be sure to run `pnpm setup-repo`.

## Architecture Rules (Must Follow)

- Do NOT import `typescript-go` or `typescript-go/internal/...` directly from Effect code.
- Always import the shim packages (`github.com/microsoft/typescript-go/shim/...`) instead.
- Shims are auto-generated by `_tools/gen_shims`. The generated output must never be edited manually.
- Before saying a request has been fullfilled and a feature complete, ensure all of the validation workflow complete without errors

## Adding Diagnostics

To add new Effect diagnostics:

1. Add the diagnostic message to `internal/diagnostics/effectDiagnosticMessages.json`
2. Run `pnpm setup-repo` to regenerate the diagnostic code from the JSON definition

The setup-repo command handles code generation for diagnostics automatically.

## Patches

All changes to typescript-go submodule should be as minimal as possible, and performed through the patches in the `_patches/` folder.
The `typescript-go/` directory is reset during check runs, so direct edits there are ephemeral unless captured in `_patches/`.
